1. Identify Core Entities (Nouns)

Ask: What “things” exist in my app?

=> Post, Autor, Reader
(Comments, User, Admin)

2. Define Relationships (This is where beginners struggle)

Ask:

Who owns what?
Can there be many?

3. Decide the Fields (Columns)

Now ask:
“What data do I actually need?”

5. Think About Real Use Cases

Ask:

Will users edit posts?
Do I need drafts?
Do I need soft deletes?

6. Start Simple, Evolve Later

This is critical.

❌ Beginners try to design the “perfect” schema
✅ Good developers design the simplest working version

add columns
refactor relationships
write new migrations
